# Easing slack-acp operator setup — plan

Living plan for the "make initial setup easy" effort. Track progress by
checking off items as they land. See chat history / `git log` for
rationale; this file is intentionally terse.

Today, a new operator needs to: build/install the binary, install an
ACP agent, paste the Slack app manifest, capture two tokens, write
config, and figure out service supervision on their own. The work
below collapses that to roughly: `brew install kfet/fir/slack-acp` →
`slack-acp init` → `slack-acp install-service`.

## Step 1 — Release infrastructure + `release` skill  *(in progress)*

Goal: `git tag vX.Y.Z && make publish` produces a GitHub Release with
prebuilt binaries for darwin/linux × amd64/arm64 (+ linux/armv6) and
auto-updates the shared `kfet/homebrew-fir` tap so `brew install
kfet/fir/slack-acp` works.

- [ ] `.goreleaser.yaml` — builds, archives (raw binaries), checksums,
      GitHub release, `brews:` block targeting `kfet/homebrew-fir`.
- [ ] `.github/workflows/release.yml` — on `v*` tag: checkout, setup-go,
      `make all`, run goreleaser. Needs `HOMEBREW_TAP_TOKEN` secret.
- [ ] `homebrew/slack-acp.rb.template` — reference copy (real one lives
      in the tap, regenerated by goreleaser).
- [ ] `skills/release/SKILL.md` — port from poe-acp, slack-acp-flavored.
- [ ] README: mention `brew install kfet/fir/slack-acp` once it exists.
- [ ] CHANGELOG entry.
- [ ] `make all` green.

## Step 2 — Operator quality-of-life

- [ ] `docs/config.example.json` + README pointer.
- [ ] Friendlier startup diagnostics for missing/invalid tokens
      (clear "go here, generate this" message; preflight `auth.test`).
- [ ] `slack-acp --print-paths` (config file, state dir, log paths,
      resolved agent command).

## Step 3 — `deploy` + `update` skills

- [ ] `skills/deploy/SKILL.md` — port from poe-acp; replace Funnel
      section with Slack app install + token capture; keep
      systemd-user + launchd recipes verbatim (env file, `set -a; .
      env; set +a` launchd wrapper, PATH gotcha, `loginctl
      enable-linger`).
- [ ] `skills/update/SKILL.md` — straight port, name-substituted.

## Step 4 — Bundle skills into the binary

- [ ] `internal/skills/` package, `go:embed all:bundle`, per-content-hash
      extract to `$TMPDIR/slack-acp-<hash>/skills/`, builtin-frontmatter
      filter, fir-style `<available_skills>` catalog.
- [ ] Inject catalog via `session.systemPrompt` cap or first-prompt
      prefix (whichever the agent advertises). Mirror poe-acp wiring.
- [ ] 100% coverage gate parity.

## Step 5 — `slack-acp init`

Interactive (or `--non-interactive` with flags) first-run wizard:

- [ ] prompt for `xoxb-` and `xapp-` tokens (or read from stdin/env);
- [ ] validate `xoxb-` via `auth.test`;
- [ ] validate `xapp-` via a short Socket Mode connect;
- [ ] write `~/.config/slack-acp/config.json` (0600) and optional
      `~/.config/slack-acp/env` (0600) for service use;
- [ ] print next-step pointer (deploy skill / `install-service`).

## Step 6 — `slack-acp install-service`

- [ ] detect platform (darwin/linux), homebrew prefix, agent command;
- [ ] write tailored systemd-user unit or launchd plist;
- [ ] `--dry-run` prints to stdout; default writes + enables;
- [ ] idempotent (won't clobber a hand-edited unit without `--force`).

## Step 7 — `slack-acp setup-app`

- [ ] embed `docs/slack-app-manifest.json` so it's available without a
      checkout;
- [ ] `slack-acp setup-app` prints the manifest + a numbered checklist
      with direct api.slack.com URLs.

## Step 8 — `slack-acp doctor`

- [ ] validate token shapes, `auth.test`, Socket Mode handshake;
- [ ] spawn agent + ACP `initialize` round-trip;
- [ ] state dir writable, agent on PATH;
- [ ] green/red checklist output.

## Step 9 — Auto-detect agent

- [ ] If `--agent-cmd` unset, probe `fir`, `claude-code --acp`,
      `gemini-cli --acp` in order; pick first on PATH.

---

## Out of scope / decided against

- **Single-token mode** — Slack Socket Mode requires both an
  App-Level token and a Bot token. Cannot collapse.
- **OAuth install link** — would require a public HTTP callback,
  defeating the Socket-Mode "no public endpoint" property.
- **Per-line `.covignore`** — never. File-level only.
